Projeto Integrador · Engenharia Biomédica

BioCore

Uma PWA que transforma o conteúdo do curso em jogos — quizzes, flashcards, cruzadas e mais — com XP, níveis e ranking entre turmas.

01O que é

BioCore é uma plataforma web (instalável como app — PWA) onde alunos de Engenharia Biomédica estudam jogando. Cada disciplina do curso vira conteúdo dentro de 9 jogos diferentes; cada partida rende XP, sobe de nível e alimenta um ranking. Professores publicam missões e acompanham a turma; admins gerenciam usuários, conteúdo e papéis.

02Stack & Infraestrutura

Aplicação full-stack em TypeScript, com renderização no servidor e banco de dados gerenciado — sem backend próprio para manter.

🖥️ Frontend

Next.js 16 (App Router) + React 19 + TypeScript. Estilos com Tailwind CSS v4. Server Actions no lugar de uma API REST separada.

🗄️ Backend & Dados

Supabase — Postgres + Auth (login/cadastro) + Row Level Security (regras de acesso direto no banco, por linha e por papel do usuário).

☁️ Deploy & PWA

Hospedado na Vercel. Manifesto de app (manifest.ts) permite instalar o BioCore na tela inicial do celular.

03Perfis & Segurança

Três papéis de usuário. O controle de acesso não vive só na tela — está reforçado no próprio banco de dados via RLS, então mesmo que alguém burle a interface, o Postgres recusa a operação.

PapelO que pode fazerOnde isso é garantido
AlunoJoga, acumula XP, entra em missões, aparece no rankingRLS: só edita o próprio perfil
ProfessorCria disciplinas e missões, acompanha desempenho da turmaRLS: policy de insert por papel
AdminGerencia usuários, papéis, disciplinas, conquistas, missões e conteúdo dos jogosServer Action com checagem de admin + RLS

06Painel Administrativo

Tudo que aparece nos jogos é cadastrado por professores/admins nessas 5 telas — nada é hardcoded no app.

👤 Usuários

Lista todos os cadastrados e permite alterar o papel (aluno / professor / admin).

📚 Disciplinas

Cadastro das matérias do curso — nome, ícone e cor de identificação.

🏅 Conquistas

Criação de badges com condição, raridade e recompensa em XP.

🎯 Missões

Missões diárias, semanais ou especiais, ligadas a um jogo e disciplina.

🧩 Conteúdo dos Jogos

Perguntas, flashcards, pares de memória, sequências de laboratório e banco de palavras.

07Telas & Navegação

Fluxo do aluno

  1. Login / Cadastro
  2. Dashboard — XP, missões ativas, atalhos de jogo
  3. Jogos → escolhe um dos 9
  4. Ranking (semanal / mensal)
  5. Perfil — nível, conquistas, histórico

Navegação

  1. Mobile: barra inferior — Home, Jogos, Rank, Missões, Perfil
  2. Desktop: sidebar fixa lateral
  3. Dark mode nativo (referência: monitor de ECG), com light mode alternável

Roteiro rápido pra quem for apresentar

  • Abre com o problema: conteúdo do curso é denso — a gamificação dá repetição espaçada sem parecer estudo forçado.
  • Mostra um jogo ao vivo (Quiz ou BioTermo são os mais rápidos de demonstrar) e o XP subindo.
  • Explica a arquitetura em uma frase: "Next.js no front, Supabase cuida de login/banco/permissões, hospedado na Vercel — sem servidor próprio pra manter."
  • Destaca a segurança: as regras de quem pode editar o quê estão no próprio banco (RLS), não só escondidas na tela.
  • Fecha com o painel admin: tudo é cadastrável — o time pode adicionar disciplinas, perguntas e conquistas sem mexer em código.
